# 掌上无畏契约 UI 设计规范整理

适用对象：炼狱 Lineup 手机 H5 / 小程序式查询工具  
参考来源：`C:/Users/Administrator/Desktop/视觉风格/` 中掌上无畏契约界面截图

## 1. 风格定位

整体气质是“游戏官方工具 + 社区内容平台”：

- 黑蓝深色用于沉浸内容、武器/商品/详情页。
- 白色浅色用于社区、消息、列表、个人战绩等高频阅读页。
- 高饱和珊瑚红作为品牌主色，用于当前状态、主操作、徽标、通知点。
- 视觉层级依赖大字号中文标题、卡片分组、胶囊标签、底部导航。
- 游戏资产图占主要视觉面积，UI 容器保持克制。

对 Lineup 工具的建议：  
查询路径页用浅色体系，点位详情页可局部使用深色图像容器，既易读又保留无畏契约气质。

## 3. 字体与字号

字体建议：

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", Arial, sans-serif;
letter-spacing: 0;
```

字号层级：

| 场景 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 页面主标题 | 26-32px | 800-900 | 1.15-1.2 |
| 顶部 Tab 标题 | 26-30px | 800-900 | 1.1 |
| 卡片标题 | 17-22px | 700-850 | 1.25-1.35 |
| 列表主文本 | 16-18px | 650-750 | 1.35 |
| 正文说明 | 14-15px | 400-550 | 1.55-1.75 |
| 标签/角标 | 12-13px | 600-750 | 1.2 |
| 数字信息 | 28-46px | 800-900 | 1 |

排版规则：

- 中文标题尽量短句，不做负字距。
- 正文使用 `text-pretty`，标题使用 `text-balance`。
- 数字使用 `tabular-nums`，比如倒计时、战绩、点位数量。
- 卡片标题最多两行，列表项标题优先单行截断。

## 4. 页面结构

### 顶部栏

掌上无畏契约常见结构：

- 顶部状态栏保留手机系统感。
- 应用头部左侧为端游标识或返回按钮。
- 中间为大标题 / 页面标题。
- 右侧为搜索、分享、设置等图标按钮。
- 当前 Tab 下方用短下划线 + 小折角标识。

Lineup 工具建议：

- 首页：左侧端游/炼狱标识，中间标题，右侧搜索。
- 流程页：左返回，中间当前步骤标题，右侧可放“帮助/来源”。
- 详情页：左返回，中间点位名称，右侧收藏或视频来源。

### 底部导航

参考图是 5 栏底部导航，中间为红色圆形发布按钮。

Lineup 工具建议如果做小工具，可简化为 3-4 栏：

- 首页
- 地图
- 最近查看
- 我的/设置

若保留参考风格：

- 中间红色圆按钮可作为“快速查询”入口。
- 当前项图标和文字使用品牌红。
- 非当前项使用 `#B6B8BD`。

## 5. 栅格与间距

移动端基准宽度：`390px`

| 场景 | 建议值 |
| --- | --- |
| 页面左右边距 | 14-16px |
| 卡片间距 | 10-12px |
| 大模块间距 | 18-24px |
| 卡片内边距 | 12-16px |
| 胶囊按钮高度 | 36-44px |
| 主按钮高度 | 48-56px |
| 底部安全区 | `env(safe-area-inset-bottom)` |

圆角：

- 入口卡片 / 内容卡片：8-12px
- 胶囊筛选：999px
- 图标底：12-16px
- 图片卡片：8-10px

阴影：

- 掌上无畏契约浅色页阴影很少，主要靠边界和留白。
- H5 原型中卡片可用轻阴影：

```css
box-shadow: 0 8px 22px rgba(18, 20, 23, 0.06);
```

## 6. 组件规范

### 6.1 胶囊 Tab

用于一级分类、地图、攻守方、包点。

默认：

- 背景：`#F4F5F7`
- 文字：`#9A9EA5`
- 高度：36-40px
- 圆角：999px

选中：

- 背景：`#FFFFFF` 或 `#FFF0F1`
- 文字：`#1C1F23`
- 可加红色通知点或红色描边

### 6.2 功能入口图标

参考消息页四宫格：

- 图标容器：64-72px 方形，圆角 14-18px
- 背景：`#F4F5F6`
- 图标：品牌红线性图标
- 文案：15-17px，字重 650-750

Lineup 可映射为：

- 地图
- 攻方
- 守方
- 最近查看

### 6.3 列表项

用于点位列表、消息列表。

结构：

- 左侧缩略图或图标：56-96px
- 中间标题 + 描述
- 右侧状态/箭头/时间

规则：

- 标题 16-18px，最多两行。
- 描述 13-14px，灰色。
- 分隔线使用 `#E8EAEE`。
- 点击热区高度不低于 64px。

### 6.4 点位卡片

Lineup 工具核心卡片建议：

- 顶部图片缩略图，比例 `16:9`。
- 标题：`A点默认包 - 防拆火`
- 标签：地图、攻方/守方、A点、用途。
- 右上角可放“防拆火”红色标签。

状态：

- 默认：白底 + 灰边。
- 点击/选中：红色描边 + 浅红底。
- 暂无数据：降低透明度，不可点击或点击进入空状态。

### 6.5 主按钮

参考商品页红色大按钮：

- 背景：`#FF4D55`
- 文字：白色，18-22px，字重 750-850
- 高度：52-56px
- 可使用斜切角增强游戏感，但低保真/工具页可先用 12px 圆角。

按钮状态：

- 默认：`#FF4655`
- Hover/按下：亮度降低 3%-6%
- 禁用：`#D8DADF` 背景，`#8A8F96` 文字

## 7. 图片与媒体规范

图片是 Lineup 工具的信息核心。

详情页顺序：

1. 火落点
2. 粘点 / 参照物
3. 瞄准点
4. 站位点
5. 操作说明 / 完整步骤

图片容器：

- 背景使用深色 `#101820`。
- 圆角 8-10px。
- 图片 `object-fit: contain`，避免裁掉 UI 信息。
- 如果是缩略图，可使用 `object-fit: cover`。

图片标题：

- 使用步骤编号，例如 `1. 火落点`。
- 标题条白底或浅灰底，保持工具感。

视频入口：

- 不建议默认自动播放。
- 可作为详情页底部按钮：`查看原视频 / 跳到 00:35`。

## 8. 图标规范

参考图标特点：

- 线性图标为主。
- 线宽偏粗，约 2px。
- 圆角端点。
- 品牌红用于当前/重要入口。
- 灰色用于普通状态。

项目执行时：

- 默认使用 Hugeicons 风格线性图标。
- 图标大小：
  - 顶部操作：24-28px
  - 底部导航：26-30px
  - 卡片内小图标：18-22px
- 图标按钮必须有 `aria-label`。

## 9. Lineup 工具页面建议

### 首页

浅色体系，强调快速查询：

- 顶部：炼狱 Lineup 查询
- 主卡：当前特工 炼狱 / 燃烧弹
- 快捷入口：选择地图、最近查看、数据来源
- 主按钮：开始查询

### 地图选择页

只显示已有数据地图：

- 地图卡：深海明珠
- 文案：已收录 1 个炼狱燃烧弹点位
- 状态：可查询

后续有更多地图时再增加卡片。

### 攻守选择页

大卡片选择：

- 攻方：有数据
- 守方：整理中

### 包点选择页

按地图实际包点展示：

- A 点：有数据
- B 点：整理中

### 点位列表页

卡片内容：

- 缩略图
- `A点默认包 - 防拆火`
- 标签：深海明珠 / 攻方 / A点 / 防拆火

### 点位详情页

更适合沉浸式图文：

- 页面标题：点位详情
- 路径标签：深海明珠 / 攻方 / A点 / 防拆火
- 图文步骤卡
- 底部可放“查看视频来源”按钮

## 10. 不建议

- 不要一上来做复杂搜索筛选，手机游戏旁查询场景更需要逐级路径。
- 不要让详情页默认裁剪游戏截图，关键 UI 和瞄点容易丢失。
- 不要使用过多渐变装饰；参考图的视觉冲击主要来自游戏图和红色品牌块。
- 不要把桌面三栏信息搬到手机端。
- 不要让暂无数据地图出现在第一版主路径里，避免玩家误以为工具不好用。

## 11. 下一步高保真方向

建议高保真采用“浅色查询流程 + 深色详情媒体卡”的混合方案：

- 查询步骤页：参考消息/发现页，白底、大按钮、胶囊筛选。
- 点位详情页：参考武器详情页，深色图片区、红色标签和强标题。
- 底部固定主按钮：参考商品页红色 CTA。
