# 试用期转正述职汇报PPT结构稿

## P1 封面
标题：试用期转正述职汇报  
副标题：UI设计师（产品方向培养）  
核心文案：姓名 + 日期；医疗信息化 / 实验室信息化（LIMS）  
页面布局建议：左侧深蓝标题区，右侧用LIMS流程节点形成科技感视觉。  
推荐图表类型：封面视觉 / 产品流程抽象图。

## P2 工作概览
标题：试用期输出覆盖多产品线，并沉淀为可复用能力  
副标题：用数据概览项目参与、产品范围、体系建设与业务支撑。  
核心文案：参与10+医疗/实验室项目；覆盖LIMS、动物中心、医院官网、硬件控制屏；搭建公司级UI组件库；参与售前支持、客户培训、标书支撑。  
页面布局建议：四个数据卡片 + 底部工作链路。  
推荐图表类型：数据卡片 / 能力链路图。

## P3 工作职责与角色定位
标题：工作重心从UI执行延伸到产品协同与业务支撑  
副标题：围绕项目目标承担设计、体系、沟通、售前与产品理解的复合角色。  
核心文案：UI设计、设计体系建设、产品协同、售前支持、项目沟通；突出“从单纯UI设计向产品思维延伸”。  
页面布局建议：中心角色定位 + 五个职责节点。  
推荐图表类型：角色定位图 / 放射图。

## P4 重点项目成果-LIMS系统2.0
标题：统一桌面端与移动端体验，支撑实验流程管理和售前演示  
副标题：围绕实验室核心场景建立设计规范，让复杂流程更可理解、更易演示。  
核心文案：桌面端、移动端；实验流程、样本管理、仪器预约、培训考核；建立统一设计规范；支持售前演示。  
页面布局建议：桌面端框架 + 移动端框架 + 实验流程横向链路。  
推荐图表类型：端侧对比 / 流程图。

## P5 重点项目成果-动物中心及医院项目
标题：多项目并行交付，覆盖管理系统、门户网站与现场终端  
副标题：在不同业务形态下保持设计一致性，同时快速适配项目交付节奏。  
核心文案：动物中心标准版、动物实验中心门户、医院LIMS、医院官网、电子门牌；突出多项目并行交付能力。  
页面布局建议：左右双泳道矩阵，中间突出“并行交付”。  
推荐图表类型：项目矩阵 / 泳道图。

## P6 硬件设备与控制系统设计
标题：形成软硬件一体化设计经验，降低屏幕落地偏差  
副标题：在小尺寸屏、控制器和设备管理系统中兼顾信息密度、操作效率与现场可用性。  
核心文案：人脸识别控制器、危化品柜管理系统、四英寸/五英寸控制屏、售货机系统；突出软硬件一体化设计经验。  
页面布局建议：四类设备屏幕卡片 + 落地关注点链路。  
推荐图表类型：设备矩阵 / 设计检查链路。

## P7 设计体系建设
标题：公司级UI组件库把一次性设计转化为长期复用资产  
副标题：沉淀颜色、字体、按钮、表单与网站设计规范，提升协作效率并降低重复成本。  
核心文案：颜色体系、字体体系、按钮组件、表单组件、网站设计规范；突出设计复用效率提升和重复设计成本降低。  
页面布局建议：左侧组件库主视觉，右侧规范模块清单，底部价值条。  
推荐图表类型：组件库架构图 / 设计系统分层图。

## P8 售前支持与项目协同
标题：设计工作向业务前端延伸，提升方案表达与客户理解效率  
副标题：参与标书、参数截图、演示视频、培训支持，让设计资产服务成交与交付。  
核心文案：标书支持、参数截图、功能演示视频、客户培训支持；突出设计之外的业务参与能力。  
页面布局建议：四阶段业务支撑流程 + 跨团队协同链路。  
推荐图表类型：流程图 / 协作链路图。

## P9 发现问题与优化方案
标题：把重复问题转化为流程资产，是下一阶段效率提升的关键  
副标题：采用“问题-原因-方案”结构，聚焦可落地机制。  
核心文案：标书截图重复劳动，对应功能截图资源库、参数映射表、AI辅助匹配；硬件屏幕设计落地偏差，对应硬件设计检查清单、开发前联合评审、真机测试机制；客户培训与售中支持重复，对应知识库、FAQ、操作视频库、问题分类机制。  
页面布局建议：三列表格，按问题行展开。  
推荐图表类型：问题-原因-方案矩阵。

## P10 个人成长
标题：从设计思维走向产品思维，能力边界持续扩展  
副标题：成长不只体现在设计工具熟练度，更体现在业务理解、客户沟通和流程优化意识。  
核心文案：理解业务场景、需求理解能力、客户沟通能力、跨团队协作能力、流程优化意识。  
页面布局建议：成长路径时间轴，突出“设计思维 → 产品思维”。  
推荐图表类型：成长路径图 / 时间轴。
